# Code of Conduct

Verso is maintained as a focused release tool for JavaScript workspaces. We want
the project to be useful, direct, and welcoming to people who report bugs, ask
questions, or contribute changes.

## Expected Behavior

- Be respectful and assume good intent while still being precise.
- Keep discussions focused on the release workflow, implementation trade-offs,
  and user impact.
- Share enough context for others to reproduce problems or understand proposed
  changes.
- Accept maintainers' decisions on scope, even when a feature is useful in a
  different project shape.

## Unacceptable Behavior

- Harassment, personal attacks, or discriminatory language.
- Repeatedly derailing issues or pull requests after maintainers have clarified
  the scope.
- Publishing private security details before maintainers have had a reasonable
  chance to investigate.

## Reporting

For conduct concerns, email `developer@univer.ai` with the subject prefix
`[verso conduct]`. For security issues, follow `SECURITY.md`.

Maintainers may edit, hide, or close comments and issues that do not follow
these expectations.
